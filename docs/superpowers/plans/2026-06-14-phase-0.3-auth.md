# Phase 0.3 — Auth Layer Implementation Plan

> Extends master spec `docs/superpowers/specs/2026-06-13-podkit-platform-design.md` §5. Built via parallel module waves + sequential integration, then one holistic review.

**Goal:** A capable auth pillar with **two first-class identities** (human users + agent tokens), built on solid primitives — password hashing, signed sessions, scoped agent tokens, an org/RBAC model — and the integration that **completes 0.2's RLS**: the authenticated session flows into Postgres session GUCs so `current_setting('podkit.user_id' / 'podkit.org_id' / 'podkit.is_agent')` drives row-level policies, injected automatically into framework loaders.

**Composed/primitive choices:** Node `node:crypto` `scryptSync` for password hashing (no external dep), HMAC-SHA256 (`createHmac`) for stateless signed tokens. Users/sessions/orgs persisted via `@podkit/db` (drizzle + pglite). No external auth lib for this slice (the primitives are small and owned); OAuth/passkeys/SSO are later sub-phases.

**New package:** `@podkit/auth` at `packages/auth`. Depends on `@podkit/db`. Consumed by `@podkit/framework` (loader session injection + GUC) and `@podkit/cli` (`podkit auth` commands).

**Conventions:** ESM, explicit `.ts`/`.tsx` import extensions, erasable-only TypeScript (no enums/namespaces/parameter properties), Node v26, pnpm; tests via `pnpm vitest run <path>`.

  src/

packages/cli/src/commands/auth.ts          # podkit auth ...             [WAVE C]
examples/hello/app/db/schema.ts            # add author_id to posts + RLS demo (optional)

## Wave A — scaffold (sequential, FIRST)
Create `packages/auth` (`@podkit/auth`), add dep `@podkit/db: workspace:*`; `pnpm install`. Commit.

## Wave B — independent modules (PARALLEL — disjoint files, unit-tested, no DB except schema defn)
- **B1 password** (`password.ts`): `hashPassword(plain): string` (scrypt, random salt, store `salt:hash` hex), `verifyPassword(plain, stored): boolean` (timing-safe compare). Test: hash→verify true; wrong password→false; two hashes of same input differ (random salt).
- **B2 token** (`token.ts`): `signToken(payload: object, secret: string): string` (base64url(json).hmac), `verifyToken(token, secret): object | null` (null on tamper/bad sig). Plus `issueAgentToken(payload, secret)` convenience returning a token with `kind:"agent"` and scopes. Test: round-trip; tampered token→null; wrong secret→null.
- **B3 rbac** (`rbac.ts`): pure checks over a membership list. `roleAtLeast(role, min): boolean` for `viewer<member<admin<owner`; `can(membership, action)` minimal mapping. Test the ordering + a couple action checks.
- **B4 schema** (`schema.ts`): drizzle tables via `@podkit/db` — `users(id, email unique, password_hash, created_at)`, `sessions(id, user_id, expires_at)`, `orgs(id, name)`, `memberships(id, user_id, org_id, role)`. Test: `getTableColumns` presence + key columns.

## Wave C — integration (SEQUENTIAL)
- **C1 core** (`core.ts`): `createAuth({ db })` → `{ signup({email,password}), login({email,password}), verifySession(token), issueAgentToken({userId,scopes}) }`. signup hashes password + inserts user; login verifies + issues a session token; verifySession returns the identity (`{ userId, isAgent:false }`) or agent identity for agent tokens. Uses password.ts + token.ts + schema.ts + the db client. Tested against in-memory pglite (real round-trip): signup→login→verifySession; bad password rejected; agent token verifies as `isAgent:true`.
- **C2 GUC + loader injection** (`guc.ts` + `packages/framework/src/loader/context.ts`): `applySessionGuc(client, identity)` runs `SELECT set_config('podkit.user_id', $1, false)` etc. so RLS policies (from 0.2) apply. Framework loader context gains `auth` (the identity) and a request-scoped db client with GUCs set. Tested: with an RLS-protected table + `ownedBy`, a query under user A's GUC returns only A's rows. **This is the RLS completion milestone.**
- **C3 CLI** (`packages/cli/src/commands/auth.ts` + register in `bin.ts`): `podkit auth signup|login|token|whoami` (token = issue agent token), `--json` envelope. Smoke against the example app.

## Wave D — finish
Full `pnpm test` green + `pnpm typecheck` exit 0; README auth section; push; PR.

## Deferred to later phases
OAuth providers, passkeys, magic links, SSO/SAML; session revocation/refresh UX; full RBAC policy matrix; the dashboard. Wave C2 wires RLS for users+agents; org-scoped GUC (`podkit.org_id`) is set but multi-org switching UX is later.
